Hexoでブログ作った時のこと２


　前回の記事からの続き。

_config.ymlの編集

　_config.ymlを編集することで、ブログのステータス(名前や時期)の変更が可能。

　このサイトの編集例
　

　-titleは、タイトル
　-subtitleは、サブタイトル(タイトルの下)
　-descriptionは、webサイトの説明
　-authorは、ブログの作者
　-languageは、言語(jaなら日本語)
　-timezoneは、時差

参考

　　・Configuration | Hexo

記事を書く

　新しい記事の追加。
　

　このコマンドを打った後に記事のファイルが追加され、名前は[記事タイトル].mdとなる。

　生成されたファイルは、source/_postに追加されている。。
　このファイルを編集することにより、記事を書くことができる。
　
　基本はMarkdown方式で書くことができ、HTMLでも可能である。

Hexoでブログ作った時のこと１

Hexo

　ブログ生成に特化したNode.js製の静的サイトジェネレータのことである。
　記事をMarkDown方式で書くことができ、Github Pagesにデプロイできる。

　簡単なコマンドで、記事の生成やブログで使えるテーマ、サイトマップ等のプラグインをインストールできる。

下記のページを参考しました。

　・所要時間3分!? Github PagesとHEXOで爆速ブログ構築してみよう！

環境構築

　(私は、Windows7でやりました。)
　・Node.jsをインストール。

　・Githubのアカウントがない場合は、アカウントを作成する。

　・Github Pagesのリポジトリを作る。

　・Githubに秘密鍵を登録し、SSHでpushできるようにする。

インストールからデプロイまで

1. Hexoをインストールをする。

2. ブログの新規フォルダの作成

3. 新しく生成したフォルダ名に移動し、必要なモジュールをインストール。

4. ローカルサーバーの立ち上げ

　http://localhost:4000/ にアクセスし、ブログが出来ているかを確認。
　ローカルサーバーを止めたい場合は、Ctrl+Cを押す。

5. 作成したブログをGithub Pagesで公開する

　gitのdeployerをインストールする。

　作成したブログのフォルダ内にある_config.ymlというファイルを書きかえることで、ブログの設定を行うことができる。

　次に、エディタを使い、_config.ymlのDeploymentを以下のように編集する。

　最後に、ブログ内にサイトをジェネレートした後、Github Pagesにアップロードする。

　エラーが何もなかった後、Github Pagesを確認すると、ブログが公開されている。
　なお、デプロイ後はタイムラグがあり、Github Pagesに公開されるまでは、1～10分くらいかかるときがある。

　デプロイしてからエラーが起こってないのに時間がたってもページが更新されない場合は、
　Githubアカウントを作成した際に使用したメールアドレスにbuildが失敗しました等のメールが入っていることもあるので、
　何が問題なのかを確認する。

追記

　私が、Windows7でHexoを導入した際に起こった問題があり、_config.yml内のlanguageをja(日本語用)にしたにも関わらず、
　ブログ内の日本語が文字化けすることがあった。原因として、ファイルを保存する際の文字コードが影響しておりました。
　ファイルを保存する際の文字コード指定がSHIFT-JISになっていた為、UTF-8に変えたところ文字化け問題は解決致しました。
　もし同じような問題が起こってしまった方は、まずはファイルを保存する際の文字コードの確認をしてみるといいでしょう。
